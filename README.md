# Clinical Trial Disease Category Classification System

---

### **Project Overview**

In clinical trial registries (such as ClinicalTrials.gov), metadata and study summaries are written in free-text format. Categorizing these studies into therapeutic areas (such as oncology, cardiology, or endocrinology) is crucial for clinical research management, resource allocation, and patient recruitment matching.

This project implements a complete, end-to-end clinical trial classification workspace:
1. **Relational Database (SQLAlchemy + SQLite)**: Stores 60,337 clinical trials structured into database models.
2. **Preprocessing Pipeline (NLTK)**: Normalizes noisy clinical jargon (lowercasing, stopword removal, regex cleaning, and lemmatization).
3. **Machine Learning Pipeline (Scikit-Learn)**: TF-IDF vectorization and Logistic Regression classification.
4. **Interactive Dashboard (Streamlit + Plotly)**: Dynamic reports, real-time trial classifier, database query tool, and explainable AI weights.

---

### **Key Features**

* **Relational Database Integration**: Stores clinical trial records in SQLAlchemy-supported databases.
* **Text Preprocessing Pipeline**: Cleans, tokenizes, filters stopwords, and lemmatizes noisy medical text.
* **TF-IDF Feature Extraction**: Engineers text features using word-level and bigram-level vectorization.
* **Balanced Classification**: Logistic Regression modeling optimized with class weighting to handle imbalanced disease categories.
* **Model Explainability (XAI)**: Extracts and displays top positive and negative text features influencing classification.
* **Streamlit Dashboard**: A dashboard for analytics, ad-hoc text classification, and database querying.

### **How It Works**

### **1. Relational Database Layer (`src/database.py`)**

The data is persisted in a relational database. By default, it is configured to use a local **SQLite** database (`clinical_trials.db`).

For production environments, you can easily point it to **PostgreSQL** by setting the environment variable:
```bash
set DATABASE_URL=postgresql://username:password@localhost:5432/clinical_trials_db
```

---

### **2. Text Preprocessing (`src/pipeline.py`)**

The pipeline normalizes unstructured text using natural language processing:
* **Lowercasing**: Unifies casing variations (e.g. *Cancer* vs *cancer*).
* **Noise Removal**: Strips HTML tags, punctuation, and numeric characters via regular expressions.
* **Stopword Removal**: Eliminates common non-informative words using the NLTK English stopwords dictionary.
* **Lemmatization**: Uses the NLTK `WordNetLemmatizer` to reduce medical terms to their base dictionary forms (e.g., *chemotherapies* $\rightarrow$ *chemotherapy*).

---

### **3. ML Modeling & Explainability (`src/train.py`)**

* **Feature Vectorization**: Uses a Scikit-Learn TF-IDF vectorizer mapping unigrams and bigrams up to a maximum of 15,000 features.
* **Classification Algorithm**: Implements Logistic Regression with `class_weight='balanced'` to offset class imbalances across the 8 categories.
* **Model Storage**: Saves the fitted pipeline and evaluation metrics to the `models/` folder.

---

