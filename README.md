# Clinical Trial Disease Category Classification System

An enterprise-grade, machine learning-driven clinical trial classification and data analytics system. The system ingests clinical study descriptions, normalizes medical text, trains a TF-IDF + Logistic Regression text classifier, and exposes an interactive dashboard and query tool.

---

## 🧬 Project Overview

In clinical trial registries (such as ClinicalTrials.gov), metadata and study summaries are written in free-text format. Categorizing these studies into therapeutic areas (such as oncology, cardiology, or endocrinology) is crucial for clinical research management, resource allocation, and patient recruitment matching.

This project implements a complete, end-to-end clinical trial classification workspace:
1. **Relational Database (SQLAlchemy + SQLite)**: Stores 60,337 clinical trials structured into database models.
2. **Preprocessing Pipeline (NLTK)**: Normalizes noisy clinical jargon (lowercasing, stopword removal, regex cleaning, and lemmatization).
3. **Machine Learning Pipeline (Scikit-Learn)**: TF-IDF vectorization and Logistic Regression classification.
4. **Interactive Dashboard (Streamlit + Plotly)**: Dynamic reports, real-time trial classifier, database query tool, and explainable AI weights.

## 🚀 Getting Started

### 1. Prerequisites & Virtual Environment

Ensure you have Python 3.10+ installed. Navigate to the project root and create a virtual environment:

```bash
# Create virtual environment
python -m venv venv

# Activate on Windows (PowerShell)
.\venv\Scripts\Activate.ps1

# Install requirements
pip install -r requirements.txt
```

### 2. Ingest the Data (ETL)

The ingestion script reads the raw CSV dataset (`data/clinical_trials_raw_patient2trial_conditions.csv`), applies text preprocessing to all 60,337 entries, and bulk-inserts them into a local SQLite database (`clinical_trials.db`).

Run the following command:

```bash
python -m src.pipeline
```

*Note: The script automatically downloads necessary NLTK corpora (`stopwords`, `wordnet`, `punkt`) if they are missing.*

### 3. Train the Model

Train the TF-IDF vectorizer and Logistic Regression classifier on the preprocessed records inside the database. It will print out evaluation metrics and save the model to the `models/` directory:

```bash
python -m src.train
```

### 4. Launch the Streamlit Web Application

Run the dashboard web application locally:

```bash
streamlit run app.py
```

Open `http://localhost:8501` in your browser to interact with the system.

## 🛠️ System Components

### A. Database Layer (`src/database.py`)
Configured to use **SQLite** (`clinical_trials.db`) by default.
To switch to **PostgreSQL** (e.g., in a production environment), set the environment variable:
```bash
set DATABASE_URL=postgresql://username:password@localhost:5432/clinical_trials_db
```
The application will automatically detect the database and build the schemas.

### B. Text Preprocessing (`src/pipeline.py`)
Cleans text through:
- **Lowercasing**
- **HTML tag stripping** via Regular Expressions.
- **Punctuation & number removal** (retains only alphabetical strings).
- **Stopword filtering** using the NLTK English stopwords dictionary.
- **Lemmatization** via the NLTK `WordNetLemmatizer` to reduce plural or conjugated medical words to their base root (e.g., *chemotherapies* $\rightarrow$ *chemotherapy*).

### C. ML Modeling & Diagnostics (`src/train.py`)
- **Feature Engineering**: A TF-IDF vectorizer mapping unigrams and bigrams, up to a maximum of 15,000 features.
- **Classifier**: Logistic Regression with `class_weight='balanced'` to handle imbalanced categories (ranging from breast cancer with 16k rows to sickle cell anemia with 1.1k rows).
- **Diagnostics**: Exports classification reports, a confusion matrix, and coefficient weights (feature importances) per class for rendering explainable predictions.
